# EEIK Adoption Model

## Purpose

Provide a structured path for organizations adopting EEIK.

Organizations should not attempt full adoption immediately.

EEIK is designed for incremental maturity growth.

---

# Maturity Model

## Level 0 — Prompt Chaos

Characteristics:

- Individual prompts
- No standards
- No governance
- No reuse

Typical Symptoms:

- Repeated work
- Inconsistent outputs
- Knowledge loss

---

## Level 1 — Shared Instructions

Capabilities:

- Shared prompts
- Shared instructions
- Basic standards

Artifacts:

```text
.github/

.claude/

README
```

Benefits:

- Initial consistency

---

## Level 2 — Capability Packs

Capabilities:

- Reusable standards
- Reusable agents
- Reusable workflows

Artifacts:

```text
capability-packs/
```

Benefits:

- Reduced duplication
- Consistent delivery

---

## Level 3 — Repository Generation

Capabilities:

- Bootstrap
- Manifest Generation
- Repository Planning

Artifacts:

```text
bootstrap/
generators/
templates/
```

Benefits:

- Faster project startup

---

## Level 4 — Governance Platform

Capabilities:

- Automated Reviews
- Governance Profiles
- Compliance Controls

Benefits:

- Reduced risk
- Improved traceability

---

## Level 5 — Knowledge Platform

Capabilities:

- Knowledge Capture
- Pattern Discovery
- Lessons Learned

Benefits:

- Organizational memory

---

## Level 6 — Engineering Operating System

Capabilities:

- Intelligent Generation
- Dynamic Agent Creation
- Continuous Improvement

Benefits:

- Enterprise-wide engineering intelligence
