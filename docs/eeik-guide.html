Getting Started

Adopting EEIK in 5 Steps

Takes under 30 minutes for any new project. Copy the config layers, customise context, then let the agents do the rest.

1

Clone & Copy the Config Layers

Fork or clone eeik_bootstrap. Copy .github/, .claude/, .vscode/, .editorconfig, AGENTS.md, and CLAUDE.md into your project root.

git clone https://github.com/<org>/eeik_bootstrap.git cp -r eeik_bootstrap/.github/ ./your-project/.github/ cp -r eeik_bootstrap/.claude/ ./your-project/.claude/ cp eeik_bootstrap/CLAUDE.md ./your-project/CLAUDE.md
2

Fill in Project Context

Edit .claude/memory/project-context.md with your actual service inventory, environments, AWS resources, and auth patterns. This is what every agent reads at session start.

# Replace all <!-- TODO --> placeholders: - Project Name, Team, Domain - Service inventory (name → repo → owner) - Environment URLs and AWS accounts - Auth patterns (Cognito, IAM roles, etc.) - Database engine, schema, ORM - CI/CD stages and triggers
3

Update applyTo Glob Patterns

Each file in .github/instructions/ has an applyTo frontmatter. Update to match your source layout so Copilot auto-applies the right standards per file type.

--- applyTo: "src/main/java/com/yourcompany/**/*.java" ---
4

Remove Unused Domain Files

Prune agents, instructions, and prompts for domains not in your stack. No mainframe? Delete mainframe agents. No Angular? Remove Angular instructions. Lean context = sharper agents.

# Example: remove IBM i / COBOL if not applicable rm .github/agents/ibmi-modernization-expert.agent.md rm .github/instructions/ibmi.instructions.md rm .claude/agents/ibmi-modernization-expert.md
5

Verify & Smoke-Test

Open a Java file, ask Copilot what standards apply. In Claude Code, run a slash command to confirm agents are loading correctly.

# In Claude Code terminal: /estimate "implement user login endpoint with JWT" # In Copilot Chat: "What coding standards apply to this file?" "Using the @architect agent, design a payment service"

Pre-Production Adoption Checklist

  • ☐ copilot-instructions.md customised with program context, stack, dependency policy
  • ☐ .claude/memory/project-context.md filled in (services, environments, auth)
  • ☐ applyTo glob patterns updated to match project source layout
  • ☐ Unused domain files removed (no mainframe → remove mainframe agents/instructions)
  • ☐ At least one Copilot agent invoked and responding in correct persona
  • ☐ At least one Claude Code slash command tested (/estimate "hello world feature")
  • ☐ .gitignore includes .copilot-*.log and session-log.md
  • ☐ Recommended VS Code extensions installed (.vscode/extensions.json)
  • ☐ Golden Rules from CLAUDE.md understood by the team
